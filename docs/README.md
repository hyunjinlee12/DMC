# docs/ — project reference material

Copy your four guideline files into this folder so the Director and agents can read
them as authoritative context:

- `DMC_Pd_workplan.md`            — WBS (T1.1–T2.8), gates G1–G4, dependencies, risks  (WHAT / order)
- `DMC_Pd_package_guideline.md`   — package / option / code snippet per task          (HOW)
- `DMC_Pd_references_summary.md`  — anchor papers, surface-model justification, benchmarks (WHY)
- `DMC_Pd_student_guide.md`       — researcher checkpoints A–E mapped to the gates

These are the source of truth for task IDs, INCAR parameters, AutoAdsorbate cutoffs,
MLIP ranking priorities, and the Shi-2024 benchmark numbers. `CLAUDE.md`, `PROJECT.md`,
and the agent prompts already bake in the critical operational details, but the agents
will consult these docs for anything not summarized there.

One-line map: workplan = the map · package guideline = the tool manual ·
references = the background · student guide = the checkpoints.
